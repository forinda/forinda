### 🛠️ What I'm building

- **[KickJS](https://kickjs.app/)** — a decorator-driven TypeScript framework for Node.js. Pluggable runtime (Express · Fastify · h3), first-party CLI with code generators, and a code-first database layer (`@forinda/kickjs-db`) with reversible migrations across Postgres, SQLite, and MySQL. <a href="https://www.npmjs.com/package/@forinda/kickjs"><img src="https://img.shields.io/npm/v/@forinda/kickjs?color=FACC15&label=npm" alt="npm version" /></a>
- **[fordb](https://github.com/forinda/fordb)** — a lean, keyboard-first desktop database client. Multi-engine (Postgres · SQLite · MongoDB planned), all free in core. Electron + TypeScript with a three-process architecture that keeps secrets in the OS keychain and previews every destructive change as SQL before it runs.
- **[nuxt-swal](https://github.com/forinda/nuxt-swal)** — a Nuxt module that drops SweetAlert2 into your app with zero setup ceremony.
- Contributions to **[TanStack Table](https://github.com/TanStack/table)**, docs fixes to **[ts-patch](https://github.com/nonara/ts-patch)**, and other paper cuts across the TypeScript ecosystem.
